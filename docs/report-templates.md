# 보고 템플릿

이 문서는 Hermes/Prime 중심 운영에서 바로 복붙해서 쓸 수 있는 최소 보고 포맷을 정리한다.

## 1. 분신 보고 템플릿

```md
- 역할:
- 수행 내용:
- 증거:
- 미검증 영역:
- Prime 판단 필요:
```

예시:

```md
- 역할: debug clone
- 수행 내용: 로그인 실패 재현, 세션 만료 로직 원인 확정
- 증거: 재현 경로, 에러 로그, failing request payload
- 미검증 영역: 운영환경 동일 증상 여부 미확인
- Prime 판단 필요: 수정 진행 여부
```

## 2. Prime 최종 보고 템플릿

```md
1. 문제
2. 원인
3. 이번에 바꾼 것
4. 검증 결과
5. 남은 리스크
6. 다음 액션
```

예시:

```md
1. 문제: 로그인 후 특정 계정에서 401이 반복 발생
2. 원인: 세션 만료 판정이 UTC 기준으로 잘못 계산됨
3. 이번에 바꾼 것: 만료 시각 계산을 KST/UTC 혼합에서 단일 UTC 기준으로 정리
4. 검증 결과: 단위 테스트 통과, 재현 시나리오 재실행 시 정상 로그인 확인
5. 남은 리스크: 운영 배포 후 기존 세션 캐시 정리 필요 가능성
6. 다음 액션: deploy clone 반영 후 실서비스 로그인 1회 확인
```

## 3. 24/7 루프 기록 템플릿

```md
- 현재 상태:
- 이번 작업 조각:
- 수행 분신:
- 실행 내용:
- 검증 결과:
- 전후 변화:
- 실패/성공 학습:
- 다음 액션:
```

예시:

```md
- 현재 상태: 메인 랜딩 LCP 느림, 최근 배포 이후 체감 성능 저하
- 이번 작업 조각: 이미지 preload 최적화
- 수행 분신: frontend clone
- 실행 내용: hero 이미지 preload 추가, 비핵심 이미지 lazy 처리
- 검증 결과: 빌드 통과, Lighthouse LCP 3.8s → 2.6s
- 전후 변화: 초기 렌더링 체감 개선
- 실패/성공 학습: preload는 핵심 이미지 1~2개에만 제한해야 효과가 유지됨
- 다음 액션: qa clone에서 모바일 회귀 확인
```

## 4. 완료 표현 가이드

분신이 쓰면 안 되는 표현:
- 완료
- 배포 완료
- 문제 해결됨
- 끝남

분신이 써야 하는 표현:
- 수정 완료, Prime 검증 대기
- 재현 완료, Prime 판단 대기
- 배포 수행 완료, Prime 실서비스 확인 대기

Prime만 쓸 수 있는 표현:
- 완료
- 조건부 완료
- 보류
- 실패
